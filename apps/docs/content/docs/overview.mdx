LegionCode is an open-source workspace for running focused coding tasks against GitHub repositories. Each task uses an isolated cloud workspace, the provider and model you select, and a review surface for inspecting every resulting file change.

<Callout variant="info" title="Private alpha">
  Hosted access is approved in limited batches. Join the waitlist from the
  [Cloud page](/cloud/); approved users sign in with GitHub to open the workspace.
</Callout>

## What LegionCode is

LegionCode combines four parts of the coding workflow in one workspace:

- **Repository context:** choose the GitHub repository and base branch for a task.
- **Model configuration:** connect a supported provider and select the model for each run.
- **Isolated execution:** run each task in its own cloud workspace and Git worktree.
- **Review:** inspect conversations, tool activity, changed files, and unified or split diffs before taking an explicit Git action.

Repository commands never execute in the browser. See [system architecture](/docs/architecture/) for service ownership, runtime boundaries, and persistence details.

## Core workflow

1. Request private-alpha access and receive approval.
2. Sign in with the approved GitHub account.
3. Select a repository and branch.
4. Connect a supported model provider and select a model.
5. Describe one concrete task in a run.
6. Review the response, changed files, and diff.
7. Continue the conversation, restore files, or perform an explicit Git action.

Runs are isolated from one another. Starting two tasks creates two independent execution contexts; it does not make two agents edit the same working tree.

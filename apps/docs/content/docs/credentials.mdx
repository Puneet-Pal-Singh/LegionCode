LegionCode uses your provider account for BYOK execution. It does not resell model access or add token markup.

## Storage model

In the hosted cloud product, provider credentials are sent to Brain over HTTPS and persisted through the cloud credential vault. Secrets are encrypted with AES-256-GCM before the encrypted payload is stored in Postgres. Records also include a non-secret fingerprint and status metadata.

<Callout variant="warning" title="Cloud credentials are not local-only">
  The hosted credential path is server-side by design. Do not describe keys as
  remaining only in the browser or on a developer machine.
</Callout>

Plaintext credentials are read only when the runtime needs to call the selected provider. API responses return connection metadata, not the secret value.

## Connect and validate

Open provider settings after GitHub sign-in, choose a provider, and enter the credential issued by that provider. LegionCode can validate supported credentials before marking the connection ready.

Validation proves that a credential can reach the relevant provider API at that moment. It does not guarantee access to every model in the provider catalog.

## Rotation and removal

To rotate a key, create the replacement with the provider, update the LegionCode connection, verify it, and then revoke the old key at the provider.

Disconnecting removes the LegionCode credential record for that provider. You should still revoke a compromised or retired credential in the provider's own console.

Never paste credentials into chat messages, repository files, screenshots, logs, support tickets, or GitHub issues.
